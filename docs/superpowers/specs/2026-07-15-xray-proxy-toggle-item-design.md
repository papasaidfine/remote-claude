# Menu item 7: xray proxy toggle for the tunnel config

Date: 2026-07-15
Status: approved

## Problem

Turning the xray `ProxyCommand` on or off for the `Host remote-claude` block
requires re-running menu item 4, which re-prompts for the server
host/user/port/reverse-port even though those values already sit in the
managed block. In the common sequence â€” item 4 first, xray (item 6) added
later â€” enabling the proxy means retyping everything.

## Goals

- A dedicated menu item that flips the `ProxyCommand` line on/off with zero
  prompts: it reuses the server values already stored in the managed block.
- Preserve the ownership invariant from the xray design: the whole managed
  block is regenerated by the script; no in-place line surgery that could
  strand or duplicate a `ProxyCommand`.

## Non-goals

- Prefilling item 4's prompts with the existing block's values (a separate,
  possibly later, improvement). Item 4's behavior is unchanged.
- macOS script only, same as the xray feature itself.

## Design

All changes live in `local/bootstrap-macos.sh`.

1. **`config_block_value <Key>` helper** â€” awk reads the value of a config
   key from between the managed BEGIN/END markers in `~/.ssh/config`.
2. **`write_ssh_config_block` gains an optional 6th parameter `force`** â€”
   when `1`, the "already contains a block, update it?" confirmation is
   skipped. Item 4 does not pass it, so its behavior is unchanged; item 7
   passes `1` so the toggle asks nothing.
3. **`run_proxy` (menu item 7)** â€” requires the managed block to exist;
   parses `HostName`, `User`, `Port`, and the reverse port (from the
   `RemoteForward` value) out of it; regenerates the block via
   `write_ssh_config_block` with the proxy flag flipped. Turning the proxy
   on additionally requires `status_xray`. The menu marker reuses the
   existing `config_proxy_on` check ([done] = proxy on). Item 4 keeps its
   `[xray]` label suffix â€” it describes the block's current content and
   matches item 7's marker.
4. **Menu wiring** â€” item 7 line, `Select [1-7, q]`, and `run_xray`'s
   closing hint points at item 7 instead of re-running item 4.

## Error handling

- No managed block: error, "run item 4 first".
- Enabling with xray unconfigured: error, "run item 6 first".
- Parse failure (block present but values unreadable): error advising to
  re-run item 4; nothing is written â€” parsing happens before any rewrite.
- The existing `~/.ssh/config` backup mechanism (`*.claude-bak-<timestamp>`)
  runs before every rewrite, unchanged.

## Testing

`test/config_proxy_toggle.test.sh`, in the style of the existing sourced
tests (`RC_SOURCED_FOR_TEST=1`), with `SSH_CONFIG` pointed at a temp file and
fake xray artifacts (`XRAY_JSON`, `XRAY_LAUNCHER`, executable
`XRAY_VENDOR_BIN`) so `status_xray` passes:

- Toggle on inserts the `ProxyCommand` line into the block.
- Toggle off removes it.
- Repeated toggling neither duplicates the line nor the block markers.
- Content outside the managed block is untouched.
- No managed block â†’ `run_proxy` fails non-zero.
- `config_block_value` returns the expected values from a written block.
